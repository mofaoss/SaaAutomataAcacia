-------------------- AI执行区（用户不要改） --------------------

你必须把“用户填写区”逐项翻译为项目代码，不可遗漏。请按以下映射执行：

[A. 用户选项 -> 代码落点]
1) 模块类型
- periodic => @module(host="periodic")
- on_demand => @module(host="on_demand")
- 不确定 => 自动判定：
  - 含“定时/每天/自动执行” => periodic
  - 含“手动开始/先站位” => on_demand

2) 入口选择
- periodic 勾选 => 生成/挂载 periodic UI
- on_demand 勾选 => 生成/挂载 on_demand UI
- 被动开关勾选 => on_demand 被动模式（trigger风格）

3) 前置条件
- 回主页=是 => run() 开头调用 back_to_home(self.auto, self.logger)
- 进入界面 => 作为状态机首状态

4) 识别方式
- OCR => auto.find_element(target, "text", ...)
- 图片 => auto.find_element("app/features/assets/<module>/<img>", "image", ...)
- 颜色块 => get_crop_form_first_screenshot + 颜色判定
- 不确定 => 先OCR，失败后图片；在注释写出依据
- 若识别方式留空：根据“识别目标”自动推断
  - 看起来是普通文本 => OCR
  - 看起来是文件名(.png/.jpg/.jpeg) => 图片匹配
  - 看起来是HSV/RGB/颜色阈值 => 颜色块

5) 动作翻译
- 鼠标点击 => 必须 auto.move_click(...)
- 按键 => auto.press_key / key_down / key_up
- 等待 => time.sleep
- 滑动 => 使用已有自动化滑动能力
- 若动作写成自然语言（如“按键 A 5秒”）：必须自动解析并转成代码，不要求用户再勾选动作类型

6) 范围与阈值
- 范围留空、未勾选、格式错误 => 默认全图（crop 不传）
- 范围合法（x1,y1,x2,y2）=> 写入 crop
- 阈值留空 => 默认 0.7

[B. 分辨率自适应（必须实现）]
1) 读取当前窗口尺寸，基准设计尺寸按 1920x1080。
2) 任何“像素坐标点击”都必须缩放：
   - scale_x = current_w / 1920
   - scale_y = current_h / 1080
   - click_x = int(base_x * scale_x)
   - click_y = int(base_y * scale_y)
   - auto.move_click(click_x, click_y)
3) 任何 crop 都优先存比例（0~1）而不是绝对像素：
   - crop=(x1/1920, y1/1080, x2/1920, y2/1080)
4) 若用户确认非16:9：
   - 优先使用比例crop + 文本OCR方式
   - 降低对模板图的强依赖
   - 在日志明确提示“非16:9可能影响模板匹配”
5) 若用户分辨率不确定：
   - 按 1920x1080 作为设计基准生成
   - 所有动作都通过缩放公式执行

[C. auto关键函数使用规范]
1) auto.take_screenshot()
   - 每轮循环最先调用，刷新当前帧。
2) auto.find_element(target, mode, ...)
   - 只负责识别，不负责最终点击。
3) auto.move_click(x, y, ...)
   - 尘白禁区唯一允许点击动作。
4) auto.press_key(...)
   - 键盘触发动作。
5) auto.get_crop_form_first_screenshot(crop=...)
   - 颜色块判断时使用。

[D. UI自动推断规则]
1) 用户没写UI => 生成最小UI：
   - CheckBox_enable
   - SpinBox_times
   - ComboBox_mode（有分支时）
   - on_demand下默认开始按钮
2) 用户提到阈值/延迟/次数/模式/按键时：
   - 阈值->Slider/SpinBox
   - 延迟->SpinBox
   - 次数->SpinBox
   - 模式->ComboBox
   - 按键->LineEdit

[E. 输出顺序（必须按此输出）]

2) usecase核心代码（声明式 @module 装饰类）
3) UI代码（若需要）

7) 若用户提供了录制脚本，追加“录制语句 -> SAA代码”映射表
